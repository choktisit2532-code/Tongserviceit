# แก้ Error PostgreSQL 42P10

สาเหตุคือ `customers.code` และ `products.sku` ใช้ partial unique index (`WHERE ... IS NOT NULL`) แต่ไฟล์ Seed เดิมระบุ `ON CONFLICT (code)` และ `ON CONFLICT (sku)` โดยไม่ใส่เงื่อนไขของ index ทำให้ PostgreSQL หา unique index ที่ตรงกันไม่ได้

ไฟล์ `seeds/001_demo_data.sql` เวอร์ชันนี้แก้เป็น:

```sql
ON CONFLICT (code) WHERE code IS NOT NULL DO NOTHING;
ON CONFLICT (sku) WHERE sku IS NOT NULL DO NOTHING;
```

## ถ้ารัน Schema ผ่านแล้ว

รันเฉพาะไฟล์ `database/seeds/001_demo_data.sql` เวอร์ชันใหม่ได้เลย ไม่ต้องลบตารางและไม่ต้องสร้างฐานข้อมูลใหม่

## ถ้ายังไม่ได้รัน Schema สำเร็จ

1. รัน `database/migrations/001_initial_schema.sql`
2. รัน `database/seeds/001_demo_data.sql`
